# Contribute

This course is intended to grow through small, reviewable contributions.

## Good Contributions

- A lab with runnable code and expected output.
- A lecture note with a concrete learning objective.
- A resource link with a reason it belongs in the course.
- A baseline comparison.
- A failure analysis.
- A reproducibility fix.

## Resource Review Checklist

Before adding a resource, answer:

- Who is this for?
- What should the learner do with it?
- Is it actively maintained or historically important?
- Is it free to access?
- Does it require unusual hardware?
- What limitation or caveat should be stated?

## Lab Review Checklist

Before adding a lab, include:

- Goal.
- Setup commands.
- Expected output.
- Baseline.
- Metric.
- Questions for reflection.
- Known limitations.

## Tone

Be precise, generous, and reproducible. Avoid hype. If a result only holds in a
toy environment, say so directly.
